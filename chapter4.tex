\subsection{数据集}

为全面评估所提方法在不同水下场景条件下的性能，本文采用两类数据集进行实验，分别覆盖动态水下场景与静态水下场景。

\textbf{（1）动态水下场景数据集}

动态水下场景实验采用UDR-GS\upcite{udrgs2024symmetry}所提供的四个数据子集，每个子集均包含若干连续帧的水下视频图像，具体包括：

\begin{itemize}
  \item \textbf{Robot}：水下机械臂操作场景，共34帧，分辨率为$800\times600$像素。机械臂的往复运动引入了规律性的刚体动态，是测试动态建模精度的典型场景。
  \item \textbf{Fish}：水下鱼类游动场景，记录了多条鱼在珊瑚礁附近自由游动的过程。鱼体的非刚性快速运动对动态建模提出了较高要求。
  \item \textbf{Coral}：珊瑚礁随水流摆动场景，呈现出软体珊瑚枝条在水流驱动下的周期性缓慢摆动，是典型的连续形变动态场景。
  \item \textbf{Streaks}：水下光线焦散/光纹场景，由水面折射形成的动态光纹投影于水底，光斑形状随时间不断变化，对场景外观建模构成挑战。
\end{itemize}

\textbf{（2）静态水下场景数据集}

静态水下场景实验采用SeaThru-NeRF数据集，包含三个典型水下拍摄地点：

\begin{itemize}
  \item \textbf{Curasao}：加勒比海库拉索岛近岸礁盘场景，水体清澈，色偏程度中等。
  \item \textbf{IUI3-RedSea}：红海水下研究站附近场景，水体可见度较高，场景包含丰富的珊瑚与礁石结构。
  \item \textbf{Panama}：巴拿马热带海域场景，水体略浑浊，散射效应相对明显。
\end{itemize}

上述数据集的组合能够有效检验所提方法在处理动态运动、水下色偏与散射等多种挑战时的综合性能。

\subsection{评估指标}

本文采用以下三项广泛使用的图像质量评估指标对各方法进行定量评价。

\textbf{（1）峰值信噪比（PSNR）}

峰值信噪比（Peak Signal-to-Noise Ratio，PSNR）\upcite{wang2004ssim}是衡量像素级重建精度的经典指标，其定义为：

\begin{equation}
  \mathrm{PSNR} = 10 \cdot \log_{10} \left( \frac{L^2}{\mathrm{MSE}} \right)
\end{equation}

其中$L$为像素最大值（对于8位图像$L=255$），$\mathrm{MSE}$为均方误差。PSNR以分贝（dB）为单位，数值越高表示渲染图像与真实图像在像素层面的偏差越小，重建精度越高。

\textbf{（2）结构相似性指数（SSIM）}

结构相似性指数（Structural Similarity Index，SSIM）\upcite{wang2004ssim}从亮度、对比度与结构三个维度综合衡量图像质量，比PSNR更接近人类视觉感知特性，取值范围为$[0,1]$，数值越接近1表示感知质量越好。

\textbf{（3）感知图像块相似度（LPIPS）}

感知图像块相似度（Learned Perceptual Image Patch Similarity，LPIPS）\upcite{zhang2018lpips}利用预训练深度神经网络提取的高层语义特征计算图像间的感知距离。与PSNR和SSIM不同，LPIPS数值越低表示渲染结果与参考图像在感知质量上越接近，更能反映人类对图像细节与纹理质量的主观判断。

\subsection{对比方法}

为全面评估所提方法的性能，本文选取以下四种具有代表性的基线方法进行对比：

\begin{itemize}
  \item \textbf{3DGS}：原始三维高斯散射方法\upcite{kerbl2023gaussian}，不包含水下物理模型，也不具备动态场景建模能力，作为基础基线。
  \item \textbf{4DGS}：四维高斯散射方法\upcite{wu2024gaussian}，通过引入时间维度扩展实现动态场景建模，但不包含水下物理退化模型，代表动态重建的主流基线。
  \item \textbf{SeaSplat}：静态水下高斯散射方法\upcite{seasplat2025icra}，将水下物理成像模型集成至3DGS框架，能够处理色偏与散射问题，但仅针对静态场景设计，代表静态水下重建基线。
  \item \textbf{UDR-GS}：基于深度正则化的动态水下高斯散射方法\upcite{udrgs2024symmetry}，同时具备动态建模与水下图像处理能力，是与本文方法最为接近的对比方法，也是本文的主要竞争对手。
\end{itemize}

上述四种对比方法覆盖了"静态通用"、"动态通用"、"静态水下"与"动态水下"四种技术路线，能够较为全面地展示各技术组件的贡献。

\section{对比分析}

\subsection{与SeaSplat对比分析}

SeaSplat\upcite{seasplat2025icra}是面向静态场景设计的水下高斯散射方法，其核心优势在于对水下色偏与散射的物理建模，但由于缺乏动态场景处理能力，其在动态数据集上的性能受到严重制约。

表\ref{tab:psnr_dynamic}中各场景的对比数据清晰地反映了这一局限。在Robot场景中，本文方法PSNR为29.71 dB，而SeaSplat仅为28.21 dB，提升约5.3\%；在Fish场景中，差距最为悬殊，本文方法（28.52 dB）相较于SeaSplat（23.30 dB）提升约22.4\%；在Coral场景中，提升幅度高达25.2\%（33.29 vs 26.58 dB）。

上述差距的根本原因在于：Fish场景中鱼体的快速游动、Coral场景中珊瑚枝条的持续摆动，以及Robot场景中机械臂的往复运动，均会导致多帧图像之间出现显著的非刚性形变。静态方法（包括SeaSplat）在处理此类输入时，无法将运动信息从场景几何中解耦，使得对应运动区域的高斯体在训练过程中接收到来自不同帧的矛盾监督信号，最终收敛至多帧叠加的模糊状态，导致渲染结果在运动区域出现严重的鬼影与模糊。相比之下，本文方法通过时间相关的形变场对运动进行显式建模，有效消除了上述模糊现象。

值得注意的是，SeaSplat在Streaks场景（26.16 dB）上的劣势相对于其他两种静态方法（3DGS为26.47 dB）更为明显，这可能与水下光焦散的时变特性同其水下物理模型的静态假设之间的冲突有关——动态光纹被错误地纳入后向散射项建模，导致模型估计出错。

\subsection{与4DGS和UDR-GS对比分析}

\textbf{（1）Robot场景}

在Robot场景中，本文方法（29.71 dB）略优于4DGS（29.49 dB）和UDR-GS（29.62 dB），三者差距在0.3\%至0.7\%之间。机械臂运动具有规律性，三种动态方法均能较好地建模其运动轨迹，因此差距相对有限。本文方法在SSIM上的优势（0.9387 vs 0.9312）表明其对机械臂边缘的结构细节刻画更为精确，但LPIPS指标（0.1587）高于UDR-GS（0.0840），反映出在感知质量方面存在一定不足，可能与本文方法引入的水下颜色恢复模块改变了局部纹理的高频特征分布有关。

\textbf{（2）Fish场景}

Fish场景是本文方法的相对薄弱场景。鱼体的快速非规律性运动与颜色恢复模块之间存在梯度冲突：在快速运动区域，形变场的估计误差会导致颜色恢复模块接收到不一致的颜色监督信号，进而干扰水体参数的准确估计。UDR-GS（29.24 dB）由于不引入颜色恢复的额外参数，其优化目标更为单一，在此类高速运动场景下反而获得了更稳定的收敛。这一结果表明，如何在快速运动场景中降低多模块间的梯度干扰，是未来工作需要重点改进的方向之一。

\textbf{（3）Coral场景}

Coral场景最充分地体现了本文方法的综合优势。珊瑚礁场景同时具有动态性（珊瑚摆动）和显著的水下色偏（蓝绿色偏移），这正是本文方法所针对的双重挑战。本文方法PSNR（33.29 dB）相较UDR-GS（32.43 dB）提升+2.7\%，相较4DGS（31.14 dB）提升+6.9\%，SSIM也由0.9475提升至0.9640。这一结果印证了完整水下物理模型在恢复珊瑚礁真实色彩方面的重要作用：4DGS缺乏色彩校正，导致颜色失真使PSNR受损；UDR-GS虽具备一定的水下处理能力，但其颜色恢复能力弱于本文方法，因此在涉及色偏的质量指标上略逊。

\textbf{（4）Streaks场景}

Streaks场景展示了本文方法最显著的性能优势。水下光焦散产生的时变光纹不仅是一种动态现象，更与水体的光学散射特性密切相关。本文方法通过动态高斯建模捕捉光纹的时变形态，同时通过水下物理模型将光焦散的颜色特性与环境散射项解耦，两个模块形成协同效应。PSNR提升1.74 dB（从28.91至30.65 dB），SSIM提升8.1\%（从0.8648至0.9349），LPIPS降低31.5\%（从0.0883至0.0605），三项指标全面领先，充分验证了动态建模与水下物理建模联合设计的有效性。

\subsection{技术路线差异分析}

本文方法与UDR-GS在技术路线上存在若干关键差异，有助于解释两者在不同场景上的表现差异。

\textbf{边缘检测策略}：UDR-GS采用Canny边缘检测算子\upcite{udrgs2024symmetry}为深度正则化提供精确的边缘定位信息，能够在纹理复杂区域提供更可靠的深度约束，但引入了边缘检测阈值等超参数，在不同场景间需要手动调整。本文方法采用基于RGB梯度的深度加权策略，通过颜色梯度的强度自适应地调节深度正则化权重，实现更为简洁的自适应机制，无需额外的超参数调节。

\textbf{深度表示方式}：本文采用DepthAnythingV2\upcite{yang2024depthv2}生成逆深度图（inverse depth map），通过逆深度表示放大近景的深度差异，使深度正则化在近景区域更为有效；UDR-GS使用直接深度（direct depth）表示。两种表示方式在深度估计误差的分布上存在差异，可能导致深度正则化在不同距离范围内的有效性有所不同。

\textbf{水下颜色恢复能力}：本文方法基于完整的Akkaynak-Treibitz修正水下成像模型，对衰减项$A(z)$与后向散射项$B(z)$均进行了显式建模与优化；UDR-GS在水下颜色处理方面采用了较为简化的策略，主要侧重于几何一致性而非颜色保真。这一差异是本文方法在涉及色偏（如Coral、Streaks场景）的指标上优于UDR-GS的主要原因。

\textbf{损失函数权重}：两种方法在深度监督损失$\mathcal{L}_\mathrm{depth}$与渲染损失$\mathcal{L}_\mathrm{render}$之间的权重分配上存在差异，这在一定程度上影响了各方法对几何精度与外观保真度之间的侧重。

\section{讨论}

\subsection{三组件互补性分析}

本文方法的核心设计理念是将三个功能模块——动态高斯建模、水下物理颜色恢复以及深度监督正则化——整合至统一框架中，三者分别针对水下动态重建问题的不同挑战维度，形成功能互补。

\textbf{动态高斯建模}的作用在于解耦场景中的静态背景与动态前景，通过时间相关的形变场对运动物体在各帧间的位姿变化进行显式建模，从而消除多帧聚合导致的运动模糊与鬼影现象。其贡献主要体现在含有明显运动的场景上，如Robot（机械臂往复运动）、Fish（鱼体游动）与Coral（珊瑚摆动）。

\textbf{水下物理颜色恢复}的作用在于校正水体对光线的选择性吸收与散射效应，使渲染的颜色外观更接近真实场景色彩而非被水体扭曲后的观测色彩。其贡献主要体现在色偏显著的场景上，如Coral（暖色珊瑚恢复）与Streaks（光焦散颜色纠正）。

\textbf{深度监督正则化}的作用在于为高斯体的三维分布提供几何先验约束，防止高斯体因优化目标的病态性而退化为平面状或出现在不合理的三维位置，从而提升几何重建的准确性与稳健性。其贡献贯穿所有场景，对几何精度指标的保证具有基础性意义。

三组件的协同效果最为集中地体现在Coral场景：本文方法PSNR为33.29 dB，相较于仅具备动态建模能力的4DGS（31.14 dB）提升+2.15 dB，相较于仅具备颜色恢复能力的SeaSplat（26.58 dB）提升+6.71 dB，均超过了单一组件所能带来的提升预期，验证了多组件联合设计的协同增益。

\subsection{局限性分析}

尽管所提方法在大多数场景上取得了较好的实验结果，但仍存在若干有待改进的局限性。

\textbf{（1）快速运动场景下的多模块干扰问题}

在Fish场景中，本文方法的PSNR略低于UDR-GS。深入分析表明，该问题的根源在于：当场景中存在高速非规律运动时，形变场的估计误差较大，而水下颜色恢复模块依赖于深度一致性假设——若动态物体区域的深度估计存在误差，则颜色恢复参数的估计也会受到干扰，两个模块的梯度在快速运动区域产生冲突。未来可考虑引入运动置信度掩码，在高速运动区域自适应地降低颜色恢复模块的监督权重，以减少两者之间的干扰。

\textbf{（2）感知质量与像素精度的权衡}

在Robot场景中，本文方法的PSNR（29.71）与SSIM（0.9387）均优于UDR-GS，但LPIPS（0.1587）显著高于UDR-GS（0.0840）。这一现象反映出本文方法在像素级精度与感知质量之间存在一定的权衡关系：水下颜色恢复模块在改善色彩保真度的同时，可能改变了局部纹理的高频统计特征，导致基于深度特征的感知距离计算结果偏高。未来可尝试在损失函数中引入感知损失项（如LPIPS损失）以缓解该问题。

\textbf{（3）不同水下场景间的超参数敏感性}

本文方法在水下物理模型的损失权重设置上对不同场景具有一定敏感性。不同水域的水体光学特性（如衰减系数、散射系数）差异较大，统一的超参数设置在部分场景上可能并非最优。如何实现水体参数的自适应估计与自动超参数调整，是未来工作的重要改进方向。

\textbf{（4）计算资源限制}

受限于笔记本端GPU（8GB显存）的计算资源，本文在高斯体数量上进行了一定的约束。在场景复杂度更高的数据集（如远景深度较大、前景物体更多的场景）上，高斯体数量的限制可能对重建细节产生不利影响。未来在更高配置的硬件平台上，有望进一步释放本文方法的性能上限。
